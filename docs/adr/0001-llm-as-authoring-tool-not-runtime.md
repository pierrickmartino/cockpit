# LLM is an authoring tool, not a runtime feature

The dashboard's structural content (actors, flows, dependency/power annotations)
is generated by an LLM **at authoring time only**: an Admin uses the generation
model to build a theme's graph, stores snapshots, and publishes a stable
version. Viewers only ever see published snapshots — the model is never invoked
at view time.

We chose this over (a) live external ingestion of the whole graph — no API
yields "where the power lies," that is editorial structure — and over (b)
generating the graph on the fly per request, which would churn a structure that
realistically changes over months, cost too much, be too slow for hovers/clicks,
and ship hallucinations straight to viewers. Authoring-time generation with a
publish gate keeps generation rare, reviewable, and correctable, while leaving
quantitative indicators free to be refreshed later by automated ingestion.
